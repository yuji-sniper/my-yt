# GitHub Issue仕様書作成コマンド

指定されたGitHub Issueの内容を読み取り、機能仕様書を作成します。

## 引数

- `$ARGUMENTS`: Issue番号

## 手順

1. 以下のコマンドでIssueの内容を取得してください:

```bash
gh issue view $ARGUMENTS --repo yuji-sniper/my-yt
```

2. Issueの内容を確認し、以下を把握してください:
- タイトル
- 説明
- ラベル（あれば）

3. Issueの内容に基づいて、以下の構成で仕様書を作成してください:

### 仕様書の構成

#### 1. 概要
- 機能の目的と背景
- 対象ユーザー

#### 2. 機能要件
- ユースケース / ユーザーストーリー
- 機能一覧（箇条書き）
- 画面・操作フロー（フロントエンドの場合）

#### 3. 技術仕様
- フロントエンド仕様（該当する場合）
  - 画面構成
  - コンポーネント設計
  - 状態管理
- バックエンド仕様（該当する場合）
  - API エンドポイント一覧
  - リクエスト/レスポンス形式
  - データモデル

#### 4. 非機能要件
- パフォーマンス要件
- セキュリティ要件
- その他制約事項

#### 5. 受け入れ条件
- 完了の定義
- テスト観点

4. 必要があればcontext7のMCPを使ってドキュメントを調べてください。

5. `.docs/specs/`ディレクトリに、`issue-{issue番号}.md`のファイル名で仕様書を保存してください。
